You see my pet project.
This is CRUD application, which allow digitalize university studying process.
Note: If you want to explore the app on your own, you can log in as a student using email: "student@gmail.com" and password: "student". Student have no rights to change or remove something, but can see all pages.

The application contains of three layers:


First layer is DAO layer. It is responsible for receiving data from database. Now it uses Spring Data JPA. But it started with JdbcTemplate and direct SQL queries to the database. Full history of application creation you can find on tab Used technologies.

Second layer is Service layer. It is responsible for the internal logic of the application, back-end validation, transaction management, and receiving data via DAO layer.

Third layer is UI layer. It is responsible for receiving HTTP requests from users and map them to relevant view (HTML page). The UI layer logic follows the REST architecture.
